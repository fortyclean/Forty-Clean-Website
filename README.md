# Forty Website

واجهة حجز وتسعير وتتبع طلبات مبنية بـ `React + TypeScript + Vite`.

## التشغيل المحلي

- تثبيت الاعتمادات: `npm install`
- تشغيل التطوير: `npm run dev`
- فحص الجودة: `npm run lint`
- بناء الإنتاج: `npm run build`

## متغيرات البيئة

انسخ `.env.example` إلى `.env` ثم عبّئ القيم.

- متغيرات الواجهة (Firebase Client): `VITE_FIREBASE_*`
- متغيرات الخادم (Tracking API): `FIREBASE_*`

## Firebase بأقل تكلفة

- استخدم خطة `Spark` المجانية طالما حجم الطلبات ما زال منخفضًا أو متوسطًا.
- لوحة الإدارة تعتمد الآن على تحميل يدوي بدل الاشتراك اللحظي لتقليل قراءات Firestore.
- أنشئ حساب Firebase Auth واحد فقط للإدارة، ولا تفتح القراءة العامة من الواجهة.
- قواعد Firestore في `firestore.rules` مقيدة بحيث:
- الزائر يرسل الطلب فقط.
- الإدارة المسجلة فقط تقرأ وتحذف وتغيّر الحالة.
- أي مسارات أخرى مغلقة افتراضيًا.
